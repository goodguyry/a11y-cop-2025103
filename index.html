HTML To The Rescue

This demo showcases HTML elements with built-in accessibility features that work seamlessly with assistive technologies and require significantly less JavaScript.

Details and Summary Elements

The <details> and <summary> elements provide a native way to create expandable content sections with built-in keyboard navigation and screen reader support.

Click to reveal more information about accessibility

This content is hidden by default and becomes visible when the summary is activated. The <details> element automatically manages the expanded/collapsed state and provides proper ARIA attributes for assistive technologies.

  • Keyboard accessible with Tab and Enter/Space
  • Screen reader announces state changes
  • No JavaScript required for basic functionality
Another expandable section

You can have multiple details elements on the same page, each maintaining their own independent state.

If the <summary> element is included, it is required to be the first element within <details>. It can be omitted.

This <details> element has no <summary>. The label is "details" and it uses the default disclosure icon.

Dialog Element

The <dialog> element represents a dialog box or interactive component with built-in focus management and modal behavior.

Modal Dialog Example

This is a modal dialog box. It traps focus within the dialog and can be closed by:

  • Pressing the Escape key
  • Clicking the close button
  • Clicking outside the dialog (if enabled)

Non-Modal Dialog Example

This is a non-modal dialog. It doesn't trap focus and allows interaction with the rest of the page.

Popover Attribute

The popover attribute designates an element as a popover element. Popovers created using the Popover API are always non-modal.

Declarative Popovers

Popover Example

This is a popover. It doesn't trap focus and allows interaction with the rest of the page. However, it will move its interactive content next in the tab order, and will return focus from its contents to the controlling <button> when closed via the Escape key.

It can be closed by:

  • Pressing the Escape key
  • Clicking outside the popover

Popover Example (Manual)

This is a "manual" popover. It uses CSS anchor positioning to keep it anchored to its controlling button. It can only be closed by clicking its controlling <button> again.

JS-Powered Popover

There is a JavaScript API for setting an element as a popover, even without a controller <button>. This page includes a "toggle" popover that is trigged by the ? key.
